A release focused on breaking boundaries and refining the essence of Quasis.

## Modular Personalization: The Theme Store Evolution

Why settle for a static experience? With v2.6, we are moving toward a fully modular UI. **QS Waveform Overlays** are now detached from the system and available as swappable components in the Theme Store ❤️ *(ctto: Saikrishna1504)*. 

But we didn't stop at waveforms. We've expanded the modular system to include **UDFPS Icons and Themes**, allowing you to personalize your fingerprint scanner visuals like never before. If you've built something cool, the [AxThemeStore repository](https://github.com/AxionAOSP/AxThemeStore_themes_repository) is now open for your PRs.

## Feel the Flow: AxAnimationEngine

Smoothness is as much about physics as it is about frame rates. This release marks the partial introduction of the **AxAnimationEngine**, bringing fluid, Material Expressive slide animations to activities and a redesigned lockscreen-to-workspace transition that makes every interaction feel more natural.

## AxBurstEngine v2.0: Pure Velocity

We've overhauled our performance framework to be smarter and more hardware-aware. This revamped framework is designed to squeeze every bit of efficiency out of your hardware while keeping things cool.

- **New Performance Framework:** Rebuilt from the ground up to handle high-demand tasks with minimal overhead.
- **Targeted Smoothness:** For devices on modern kernels (**5.X with uclamp support**), our tests show significantly smoother scrolling in heavy apps and snappier transitions across the system.
- **Cluster Awareness:** Specifically optimized for mid-range chipsets (6:2 cluster types). Even with heavy blurs enabled, your UI remains responsive and fluid.
- **BSP-Level Boosting:** Deep integration with the Board Support Package (BSP) allows tuning that is hardware-aware.
- **Dynamic Resource Mapping:** Uses precise "Boost Opcodes" to scale CPU and GPU resources exactly when you need them.
- **Smart Fallback:** Even without custom device-specific configuration, the engine gracefully falls back to a highly-optimized generic approach—ensuring every device feels fast and stable.

## Security & Privacy: The Sandbox Vault

Security should never be an afterthought. We're introducing the **Sandbox "Vault"** *(ctto: Saikrishna1504)*
—a new encrypted storage space within the Sandbox environment. You can now hide and protect your most sensitive files with industry-standard encryption, accessible only within the Sandbox app.

## Refined Across the Board

Every corner of AxionOS has been polished to ensure "Quasis" is our most stable and cohesive release yet.

### System UI & Media
- **Dynamic Bar & Status Bar:** We've overhauled the Dynamic Bar with a fresh new lockscreen media layout and removed redundant notification event overrides. The pipeline is now connected to the default status bar chip—meaning call chips show the AOSP UI and recording uses the default system interface.
- **New QS Tiles:** Toggle your **Routines** and **DNS settings** instantly with new, dedicated Quick Settings tiles.
- **AxionFx Audio Engine:** Updated to support the latest hardware, including fixes for newer audio AIDL on devices like the Pixel 8, 9, and 10 series *(ct: Corazon404)*.

### Workflow & Productivity

- **AxPcMode:** Desktop mode is better than ever with significant layout fixes and UI improvements.
- **Pulse Launcher:** Improved profile handling for Work and Private spaces, alongside new smart drawer organizational tools.
- **Gamespace:** A complete UI/UX overhaul for a more immersive experience.
- **Edgelauncher:** Added a default launch mode setting *(ct: Saikrishna1504)* and various responsiveness fixes.
- **Routines:** Expanded automation capabilities with new **HTTP/Internet Routines**.

### System Stability

- **Power Management:** Deep doze optimizations and stability fixes *(ct: bijoyv9)*.
- **SoC Optimization:** Targeted performance fixes for mid-range chipsets like the MTK Dimensity 7200 Pro.
- **Bug Squashing:** Resolved lags in Unity-based games and addressed long-standing issues in the App Lock and Keybox modules.
